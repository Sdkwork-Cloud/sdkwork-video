# Docs Directory

## Purpose
Repository/application documentation, architecture decisions, runbooks, design notes, changelogs, and user/developer guides.

## Owner
Documentation teams and technical writers.

## Allowed Content
- Architecture decision records
- Runbooks
- Design notes
- Changelogs
- User guides
- Developer guides

## Forbidden Content
- Implementation code
- Generated SDK output
- API contracts
- Secrets or credentials

## Related Specs
- `DOCUMENTATION_SPEC.md`
- `ARCHITECTURE_DECISION_SPEC.md`
- `GOVERNANCE_SPEC.md`

## Verification
- Documentation is up-to-date
- ADRs follow template
- Runbooks are tested
