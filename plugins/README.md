# Plugins Directory

## Purpose
Application/runtime plugin source packages, marketplace plugin implementations, or extension packages.

## Owner
Plugin developers and extension maintainers.

## Allowed Content
- Plugin source packages
- Plugin manifests
- Plugin documentation
- Plugin tests

## Forbidden Content
- Generated SDK output
- API contracts
- Runtime configuration
- Secrets or credentials

## Related Specs
- `APPLICATION_SPEC.md`
- `MODULE_SPEC.md`
- `COMPONENT_SPEC.md`

## Verification
- Plugins have proper manifests
- Plugins follow naming conventions
- Plugins do not vendor unrelated external toolchains
